# Implementation Blueprint: Nationwide 4-Level Syllabus Scraper

## Context
- **Goal**: Webシラバスの最上位（全国高専一覧）から4階層を自動でクローリングし、ユーザーの所属（高専名・学科・学年）に合致する科目情報と評価割合を抽出してJSON化する。
- **Target Files**:
  - `backend/services/syllabus_scraper.dart` (または Pythonのスクリプト等)

## Step-by-Step Instructions

### Step 1: 学校の特定 (Level 1: 学校一覧)
1. シラバスサイトのトップページ（学校一覧）のHTMLを取得してください。
2. 引数の `kosenName`（例: "長野工業高等専門学校"）に完全一致、または部分一致するボタン（`<a>`タグ等）を探してください。
3. その高専の専用ページへのリンク（`href`）を抽出してください。

### Step 2: 学科・コースの特定 (Level 2: 学科一覧)
1. Step 1 で取得したURLにアクセスし、該当高専の学科一覧HTMLを取得してください。
2. 引数の `courseId`（例: "情報科学・工学系"）を含む学科ブロックを特定してください。
3. そのブロック内にある「本年度の開講科目一覧」のリンク（`href`）を抽出してください。

### Step 3: 開講科目と詳細リンクの抽出 (Level 3: 科目一覧)
1. Step 2 で取得したURLにアクセスし、科目一覧テーブルを行ごとにループ処理してください。
2. 引数の `grade`（学年）に該当する列をチェックし、単位数（または履修フラグ）が存在する行のみを抽出対象とします。
3. 抽出対象の行から以下を取得してください：
   - `subjectName`: 授業科目名
   - `detailUrl`: 科目詳細ページへのリンク（絶対URLに変換すること）
   - `credits`: 単位数
   - `teacher`: 担当教員

### Step 4: 評価割合の抽出 (Level 4: 科目詳細)
1. Step 3 で抽出した各科目の `detailUrl` にループ処理でアクセスし、詳細ページのHTMLを取得してください。
2. 詳細ページ内の「評価割合」テーブルを解析し、以下の数値を抽出してください：
   - `exam`: 「試験」の数値
   - `assignment`: 「発表」「ポートフォリオ」「その他」などを合算した数値
3. これらを Step 3 の基本情報と結合してください。

### Step 5: JSONレスポンスの生成
1. 最終的なデータを `GET /api/v1/syllabus` のレスポンススキーマに沿った JSON (List形式) に変換して返却してください。

## Constraints
- **Rate Limiting (重要)**: 相手サーバーへの負荷を防ぐため、Step 4 のループ処理（科目ごとの詳細ページアクセス）の間には、必ず `1秒` 以上のスリープ（Delay）を入れること。
- **Robustness**: リンクが見つからない、またはテーブル構造が一部異なる科目があった場合は、`try-catch` でエラーを補足し、その科目はスキップするかデフォルト値を入れて後続の処理を継続すること。
