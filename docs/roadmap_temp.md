# Phase 4: シラバスの自動取得と成績管理の統合ロードマップ

## Context
- **Goal**: ユーザーが所属（高専・学年・コース）を選択するだけで、公式シラバスから履修科目、単位数、評価割合を自動取得し、成績管理（点数入力・単位計算）を可能にする。
- **Core Concept**: 「オフラインファースト」を維持しつつ、サーバー側でスクレイピングしたデータをローカルDBに同期するハイブリッド方式。


### 1. ユーザープロフィールの拡張と設定画面
ユーザーがどのシラバスを取得すべきか特定するための情報を保存・管理します。
- [x] **Data Model拡張**: `User` モデルに `kosenName`, `grade`, `courseId` フィールドを追加。
- [x] **所属設定UIの作成**: プロフィール画面または初期設定フローに、高専・学年・コースを選択するドロップダウンを実装。
- [x] **永続化対応**: 選択された所属情報をローカルDBに保存。

### 2. バックエンド：シラバス解析エンジンの構築
安定性と保守性のために、スクレイピング処理をサーバー側に集約します。
- [ ] **Scraper実装**: Python (BeautifulSoup/Selenium) 等を用い、特定高専のシラバスサイトから科目名・教員・単位・評価割合を抽出。
- [ ] **APIエンドポイント作成**: `GET /api/v1/syllabus` を作成し、所属情報を受け取って解析済みJSONを返す。
- [ ] **キャッシュロジック**: 同一条件の検索結果をデータベースに保存し、リクエストの高速化と相手サーバーへの負荷軽減を図る。

### 3. アプリ：科目データの一括インポート
サーバーから取得した詳細な科目データをローカルDBへ反映します。
- [ ] **APIクライアント実装**: サーバーからシラバスJSONを取得する通信処理（Repository層）。
- [ ] **一括保存（Upsert）処理**: 取得したデータを `Course` テーブルへ保存。既に手動入力されたデータがある場合の競合解決ロジック。
- [ ] **評価割合の正規化**: 「試験 70%, その他 30%」といった文字列を数値データとしてDBのカラムに格納。

### 4. 成績管理・シミュレーション機能
取得した「評価割合」データを活用し、実用的な計算機能を提供します。
- [ ] **点数入力UI**: 各科目詳細画面に「中間」「期末」「レポート」等の点数入力欄を作成。
- [ ] **暫定評価算出ロジック**: `(入力点数 × シラバスの割合)` を計算し、現在の合計点をリアルタイム表示。
- [ ] **単位取得シミュレーター**: 合計60点に到達するために「次のテストで何点必要か」を表示するガイド機能。

---

## ⚠️ 制約と注意点（Constraints）
1. **サーバー仲介の原則**: アプリから直接Webサイトをスクレイピングしない（サイト構造変化への対応を容易にするため）。
2. **オフライン動作**: 一度取得したシラバスデータは必ずローカルDBに保存し、圏外でも成績入力・閲覧を可能にする。
3. **エラーハンドリング**: シラバスが取得できなかった場合でも、手動で科目を追加できるバックアップ手段を残す。
